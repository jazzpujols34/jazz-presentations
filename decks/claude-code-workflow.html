The System

Six files. Total recall.

Session Start
CLAUDE.md + soul.md
Auto-loaded: architecture, standards, personality, guardrails
During Session
Skills/ + Knowledge/
Decision frameworks and technical memory, on demand
Cross-Session
DASHBOARD.md
Priorities, blockers, and project status across all sessions
Session End
HANDOVER.md
Write the relay baton -- next session picks up exactly here
Cycle repeats: read handover, delete, work, write new handover
Auto-Loaded Context

CLAUDE.md + soul.md are read at every session start, providing project rules and communication style.

On-Demand Enrichment

Skills and knowledge files are loaded only when relevant, keeping context lean.

Persistent State

DASHBOARD.md tracks priorities across sessions, updated by the PM system.

Relay Protocol

Each agent writes a HANDOVER.md baton for the next, ensuring zero context loss.

Key Principles

The rules that compound

Every correction = a new rule

No mistake happens twice. Each fix gets codified as a Learned Rule in CLAUDE.md.

Know what NOT to build

Auth goes to Clerk. Payments to Stripe. Files to R2. Only build the core value prop.

Challenge the problem first

Delete, simplify, speed up, automate — in that order. Question before you code.

Ship beats perfect

Core value prop works + 3 users can complete the flow = ship it. Iterate in production.
